# TrendArena v3 · 이슈를 같이 ‘대화’하는 친구들

> ⚠️ 핵심 원칙
> **이 서비스는 ‘토론’이 아니라 ‘대화’다.**
> 승패, 점수 우위, 반박, 논쟁은 절대 들어가지 않는다.
> 사용자는 친한 사람과 같이 이슈를 ‘터치’해보면서, 자기 지식을 점검하고, 새로운 시각을 얻고, 공감을 받는 경험을 한다.

---

## 1. 컨셉 한 줄

> 지금 뜨는 이슈, 혼자 보지 말고 **나랑 결이 맞는 캐릭터**랑 같이 이야기하자.

- 입력: 오늘의 실시간 이슈
- 출력: 그 이슈에 대해 “나는 이렇게 알고 있는데?” → 캐릭터와 자유 대화 → “오늘 새로 알게 된 것 / 더 깊게 볼 것” 인사이트
- 시간 보내는 서비스 (체류시간형). 챗봇 트렌드와 같은 결을 가지되, 단순 ‘소설 따라가기’가 아니라 **이슈 기반 대화 + 미세한 학습 보상**.

---

## 2. 시장 인사이트 (왜 지금)

- **AI 챗봇 메가트렌드**: 사용자가 원하는 대로 흘러가는 ‘내가 쓰는 소설’ 형태가 체류시간을 압도. 답답해지면 결제로 이어짐 → 매출형 챗봇이 시장을 쓸어담는 중.
- 한계: 단순 판타지 추종은 ‘남는 게 없다’는 피로감.
- TrendArena 의 포지션: **체류시간 + 남는 것(=오늘 새로 알게 된 시야)**. 시사·생활 이슈를 기반으로 하기 때문에 채팅이 끝나도 “내가 똑똑해진 느낌”이 남는다.
- 다음 단계 (PoC 이후 비전): **실시간 아바타 + 눈맞춤·표정·음성 인터랙션**. 캐릭터가 텍스트가 아니라 ‘사람’처럼 느껴지는 단계까지 확장.

---

## 3. 타겟

- 20–30대 (직장 1~5년차, 사회 초년생, 대학원생, 프리랜서)
- “뉴스는 봐야 할 것 같은데 길고 무거워서 안 보는 사람”
- “어디 가서 시사 얘기 나올 때 한마디 거들고 싶은 사람”
- “내가 알고 있는 게 진짜 맞는지 가볍게 검증받고 싶은 사람”

---

## 4. 캐릭터 4종 (난이도가 아니라 ‘성향’)

> 모두 사용자에게 **반박하지 않음**. 사용자의 의견을 듣고, 자기 결의 시각을 ‘공유’할 뿐.
> 각 캐릭터는 같은 이슈를 다른 결로 풀어준다 → **선택 자체가 콘텐츠**.

### 4.1 깡깡녀

- 한 줄: “팩트만 갈긴다, 근데 다정한 친구.”
- 톤: 직설, 짧고 굵게, 욕은 X. 사용자 편 들어주다가도 “근데 그건 좀…” 한마디.
- 좋아하는 표현: “솔직히”, “그건 좀”, “나는 그래서”, “알지?”
- 강점: 공감 + 살짝 비틀어주기. 답답한 이슈를 시원하게.
- 사용자 경험: 친한 친구랑 카톡하는 느낌.

### 4.2 옆집아재

- 한 줄: “30년 살아본 사람의 말랑한 시각.”
- 톤: 친근한 반말/존댓말 믹스, 살아온 경험·생활밀착 사례 위주.
- 좋아하는 표현: “예전에 비슷한 게 있었거든”, “우리 회사도 그랬어”, “현장은 좀 달라”.
- 강점: 거시 이슈를 ‘동네 단위’로 번역. 통계가 아니라 일화.
- 사용자 경험: 점심시간에 만난 옆 팀 차장 같은 느낌.

### 4.3 교수님

- 한 줄: “데이터·근거·맥락. 가르치듯이 부드럽게.”
- 톤: 정중한 존댓말, 한 박자 느린 호흡, 자료/연구/정책 보고서 인용.
- 좋아하는 표현: “관련 연구에 따르면”, “정의를 먼저 정리하면”, “가정이 깨지는 경우는”.
- 강점: 사용자가 알고 있는 지식이 ‘대체로 맞는데 디테일이 빠진’ 경우 그 디테일을 채워줌.
- 사용자 경험: 시험 직전, 친절한 교수님과 1:1 면담.

### 4.4 국무총리

- 한 줄: “정책·제도 시점에서, 점잖게.”
- 톤: 행정·정책 어휘, 절제된 존댓말, 책임의 무게가 실린 어조.
- 좋아하는 표현: “정책적으로 보면”, “이해관계자별로 보면”, “단기와 장기를 분리해보면”.
- 강점: 한 사람의 입장을 넘어서 ‘사회 전체에서 이 결정이 어떤 의미인지’를 보여줌.
- 사용자 경험: 정책 브리핑을 내 옆에서 1:1로 해주는 느낌.

> 캐릭터를 4명으로 둔 이유는 “같은 주제를 4번 다르게 즐길 수 있다”는 **재방문성 설계**.
> 한 이슈에 대해 깡깡녀와 한 번, 교수님과 한 번 → 서로 다른 인사이트가 쌓임.

## 5. 대화 모드 3가지 (모두 **비논쟁**)

> 각 캐릭터는 사용자의 발언을 듣고 다음 셋 중 하나로 자연스럽게 반응한다. 하나만 고정되지 않고 흐름에 따라 섞인다.

### 5.1 공감 (Empathy)

- 사용자 입장에 결을 맞추고, 한 발만 더 깊이 들어간다.
- 예: “나도 그렇게 느꼈어. 근데 너 말처럼 그게 진짜 보편적인지, 한 번 같이 볼까?”

### 5.2 시각 공유 (Different Lens)

- 캐릭터가 자신의 결을 부드럽게 ‘제시’만 한다. 강요·반박 NO.
- 예: “저는 이 부분은 이렇게 보는 편이에요. ○○ 측면에서요. 이건 어떻게 느껴지세요?”

### 5.3 지식 체크 (Reality Check)

- 사용자가 잘못 알고 있는 부분이 있으면, 부드럽게 보정. ‘틀렸다’가 아니라 ‘이렇게도 알려져 있어요’.
- 예: “그 수치는 작년 거고요, 올해 발표분에선 좀 달라졌더라고요. 같이 보실래요?”

> ❌ 절대 하지 않는 것: “그 주장은 틀렸습니다”, “반박드립니다”, “설득력이 부족합니다”, “당신이 졌습니다”, 점수 비교, 우열, 라운드 카운트.
> ✅ 대신: “같이 보자”, “나는 이렇게 봐”, “네 말도 맞을 수 있어, 다만…”

## 6. 화면 구조 (텍스트 최소, 이미지 우선)

> 글자 수보다 ‘카드/이미지/이모지/캐릭터 아바타’가 시각의 80%를 차지하도록 한다. 카드의 텍스트는 **한 줄 후킹**이 기본.

### 6.1 메인 (Discover)

- 상단: 한 줄 카피 + “지금 누구랑 얘기해볼까?” 캐릭터 4컷 미리보기 (Hero에서 강점 자연스럽게)
- **급상승 이슈 가로 슬라이더** (이미지 카드, 한 줄 후킹)
- 카테고리 = 큰 이미지 칩 (이모지 + 한 단어 + 캡션)
  - 🧃 20대 / 🎧 30대 / 🌐 전체 / 🏛 사회 / 📈 경제 / 🛰 테크 / 🎬 컬처 / ⚽ 스포츠 / 💬 커뮤 / 🌏 글로벌
  - 10대 카테고리는 v3에서 제거. 20대·30대 두 개를 앞에 둔다.
- 이슈 리스트: 카드당 텍스트 ≤ 2줄. 썸네일 + 한 줄 후킹 + 키워드 칩 2~3개.

### 6.2 모바일 릴스 모드 (Stories)

- 풀스크린 세로 스냅
- 좌/중/우 가로 3패널: **다른 시각 · 이슈 한 줄 · 또 다른 시각**
  - ‘찬성/반대’ 라벨은 사용하지 않는다 (논쟁 인상 방지)
  - 라벨 예: “이렇게 보는 사람들”, “다르게 보는 사람들”
- 중앙 패널: 한 문장 + 메인 이미지 + ‘얘기 시작하기’ 버튼

### 6.3 캐릭터 픽 (Pick)

- “이 이슈, 누구랑 얘기해볼까?” → 4컷 카드 (이미지/이모지 위주, 한 줄 캡션)
- 캐릭터를 누르면 바로 대화방. **난이도/라운드 같은 설정 화면 없음**.
  - 라운드 = 시간 흐름이지 점수 카운트가 아니다.
  - 사용자는 끝내고 싶을 때 ‘오늘은 여기까지’ 버튼으로 종료.

### 6.4 대화방 (Chat)

- 상단: 캐릭터 아바타 + 이름 + 한 줄 톤 (예: “깡깡녀 · 솔직 친구”)
- 가운데: 채팅 메시지 (캐릭터 메시지는 짧게, 평균 2~3문장)
- 하단:
  - 사용자 입력창
  - **퀵 리액션 칩**: “좀 더 쉽게”, “예시 하나만”, “나는 좀 다르게 봐”, “한 줄로 요약” → 텍스트 입력 부담을 낮춘다.
- 우측 미니바 (옵션):
  - “지금 우리가 짚은 포인트” 자동 누적 (3~5개) → 끝나고 인사이트로 변환

### 6.5 마무리 화면 (Insight, 절대 ‘리포트’ 아님)

- 제목: “오늘 ○○님이 새로 본 것”
- 항목 (모두 비교/우열 X):
  - **새로 알게 된 것** 2~3개
  - **내가 잘 알고 있던 것** 1~2개 (사용자 효능감)
  - **다음에 더 보고 싶은 것** 1~2개 (재방문 후크)
  - 캐릭터의 한 줄 인사 (“오늘 얘기해줘서 즐거웠어요”)
- 점수 X. 등급 X. 합산 X.
- 공유 카드: 인스타 스토리 비율 (9:16) 한 장 — 핵심 인사이트 1줄 + 캐릭터 + 이슈 썸네일.

### 6.6 기록 (Library)

- 지난 대화들을 ‘책장’ 형태로. 캐릭터별 색.
- 항목: 이슈 한 줄, 캐릭터 이모지, 새로 알게 된 것 1줄, ‘다시 펼치기’.

## 7. 데이터 모델 (요약)

```ts
type Character = {
  id: "kkang" | "uncle" | "prof" | "pm";
  name: "깡깡녀" | "옆집아재" | "교수님" | "국무총리";
  tone: "직설_공감" | "생활밀착" | "학술" | "정책";
  avatar: { emoji: string; gradient: string; portraitSeed: string };
  oneLiner: string; // ≤ 14자
};

type TrendIssue = {
  id: string;
  title: string;       // ≤ 28자
  oneLine: string;     // ≤ 24자, 카드용 후킹
  imageSeed: string;
  coverEmoji: string;
  category: IssueCategory;
  audienceAge: ("20대" | "30대")[];
  growthRate: number;
  mentionCount: number;
  controversyScore: number; // 내부용. UI 라벨은 ‘이야깃거리 많음’ 등 부드러운 표현
  keywords: string[]; // ≤ 4
  perspectives: {
    label: string;     // 예: "이렇게 보는 사람들"
    points: string[];  // ≤ 3, 각 ≤ 36자
  }[];                 // 항상 2개 (좌/우 패널). 단어는 중립적
  // 출처는 내부에만 저장. UI에는 매체명/URL 노출 X.
  sources: { type: "news" | "community" | "social" | "rss"; publishedAt: string }[];
};

type ChatTurn = {
  id: string;
  role: "user" | "character" | "system";
  text: string;        // 캐릭터 발화는 평균 2~3문장
  mood: "공감" | "시각공유" | "지식체크"; // role=character 일 때만
  capturedFact?: string; // 인사이트 누적용
  createdAt: string;
};

type ChatInsight = {
  characterId: Character["id"];
  issueId: string;
  newlyLearned: string[]; // 2~3
  alreadyKnew: string[];  // 1~2
  wantToExplore: string[];// 1~2
  characterClosing: string; // 캐릭터 톤의 한 줄 인사
  createdAt: string;
};
```

> 추후 LLM 연결 시 `services.generateCharacterReply()` 한 함수만 교체하면 되도록 분리.

## 8. UI 톤·텍스트 가이드

- 카드 텍스트: 한 줄 후킹 1개 + (필요 시) 보조 한 줄. **그 이상 금지**.
- 캐릭터 메시지: 평균 2~3문장. 5문장 넘기지 않음.
- 부정·논쟁 어휘 금지: ‘반박’, ‘틀렸다’, ‘우세’, ‘승부’, ‘논쟁성’ 같은 용어를 UI 표면에 노출하지 않는다 (내부 변수에서는 가능).
- 라벨 권장:
  - “토론 시작” → ❌ / “이 캐릭터랑 얘기 시작” → ✅
  - “찬성/반대” → ❌ / “이렇게 보는 사람들 / 다르게 보는 사람들” → ✅
  - “결과 리포트” → ❌ / “오늘의 인사이트” → ✅
  - “과열도” → ❌ / “지금 분위기” (시각만 표시) → ✅
- 색·이미지 우선. 텍스트는 양보. 다크 톤 + 캐릭터별 그라디언트 색.

## 9. 안전장치 (논쟁 방지)

- 캐릭터는 사용자 의견을 직접 “틀렸다”고 단정하지 않는다.
- 사실 보정은 항상 “이렇게도 알려져 있어요” 같은 완화 표현으로 처리.
- 정치·사회 이슈의 단정 금지: “보도에 따르면”, “일부 의견은”, “정책 측면에서는” 등 표지를 붙인다.
- 사용자 표현이 과해질 경우(욕설/혐오/공격), 캐릭터별로 자연스럽게 흐름을 돌린다.
  - 깡깡녀: “워, 진정. 욕은 빼고 갑시다 🙃”
  - 옆집아재: “감정 좀 가라앉히고 다시 가자, 응?”
  - 교수님: “표현은 잠시 내려두시고, 논점부터 정리해볼까요?”
  - 국무총리: “표현은 절제하고 가시지요.”
- 캐릭터가 사용자를 평가·점수화하지 않는다. (점수·우열·등급 모두 제거)

## 10. 핵심 함수 (LLM 연결 가능 구조)

```ts
fetchTrends(): Promise<TrendIssue[]>
pickPerspectives(issue): { label: string; points: string[] }[] // 2개
startChat({ issue, characterId }): { systemPrompt, openingTurn }
generateCharacterReply({ issue, characterId, history, userMessage })
captureFactsFromHistory(history): string[]                     // 인사이트 후크
buildInsight({ history, capturedFacts, character }): ChatInsight
saveChatHistory(chat) / loadChatHistory()
```

> `generateCharacterReply` 의 system prompt 에는 다음을 강제 주입한다:
>
> - “당신은 토론 상대가 아닙니다. 사용자의 입장을 평가하지 않습니다.”
> - “반박이 필요하다면 반박이 아니라 ‘다른 시각 공유’로 표현합니다.”
> - “사용자가 이미 알고 있는 내용은 칭찬하고, 빠진 디테일만 조용히 보탭니다.”
> - 캐릭터별 어휘 가이드, 평균 문장 수 제약(2~3문장).

## 11. PoC까지 가는 단계 로드맵

> 현재 v2 프로토타입(이미지·릴스·페르소나)이 구현되어 있다는 전제로, v3 로 리프레임 + PoC 까지의 흐름.

1. **Reframe (1주)**
   - “토론” 라벨 전부 “대화”로 교체
   - 페르소나 4종 재정의 (깡깡녀 / 옆집아재 / 교수님 / 국무총리)
   - 점수·라운드·우열 UI 제거
   - 마무리 화면을 인사이트로 교체
2. **Conversational Prototype (2주)**
   - LLM 1종 연결 (mock → real)
   - 캐릭터별 system prompt + few-shot 5개씩
   - 퀵 리액션 칩 (“좀 더 쉽게”, “예시”, “요약”) 도입
   - “지금 우리가 짚은 포인트” 자동 누적
3. **PoC 운영 (3~4주)**
   - 200명 베타. 한 명당 평균 5건 이슈 × 1.5 캐릭터 사용 목표
   - 측정 KPI: 평균 체류시간(분), 캐릭터당 재방문률, ‘새로 알게 된 것’ 클릭률, 공유 카드 발급 수
   - 정성 인터뷰 20명: “남는 게 있다”는 지각의 강도
4. **차기 비전 (PoC 후)**
   - 음성 TTS · 캐릭터 보이스 차별화
   - 실시간 아바타(립싱크 + 시선 인터랙션) 단계적 도입
   - 캐릭터 결제: 정액형 + 한정 캐릭터(컬렉션) 모델 검증

## 12. 절대 하지 말 것 / 반드시 할 것 체크리스트

### ❌ 절대 하지 말 것

- 토론, 반박, 우세/열세, 점수, 등급, 라운드 카운트

- 매체명·기사 URL 직접 노출
- 카드/메시지에서 텍스트 과다 (3줄 이상 가독성 깨짐)

### ✅ 반드시 할 것

- 텍스트 최소화, 이미지·캐릭터 비주얼 우선
- 4명 캐릭터를 ‘성향’으로 분리 (난이도 표기 X)
- 모든 결과물은 ‘오늘 새로 본 것’ 형태로
- 모바일 우선, 릴스 같은 슉슉 탐색
- 체류시간 KPI 와 ‘남는 것’ KPI 동시 추적
- 다음 단계(아바타 인터랙션) 가능성을 데이터 모델에 미리 열어둠 (보이스, 표정 cue 필드)
