너는 시니어 풀스택 프로덕트 엔지니어이자 UX 디자이너다.

나는 아래 컨셉의 서비스를 만들고 싶다. 먼저 MVP 프로토타입을 만들고 싶다. 설명만 하지 말고, 바로 실행 가능한 수준의 프로토타입 코드를 만들어줘.

서비스 컨셉:

- 트렌드, 이슈들을 크롤링하고 정제해서 각 타입별 실시간 이슈, 유행, 트렌드를 보여주는 서비스
- 과거 네이버 실시간 검색어처럼 “지금 사람들이 무엇에 관심을 갖는지”를 한눈에 보여주는 느낌
- 단순히 이슈 목록만 보여주는 것이 아니라, AI가 일부 주제를 가져와 사용자와 토론을 진행
- 토론 구조는 AI vs 사용자
- 사용자는 자신의 의견을 말하고, 토론 AI는 반대 입장 또는 균형 잡힌 반론을 제시
- 논쟁이 끝나면 판단 AI가 중립적으로 전체 대화를 분석하고 사용자에게 피드백을 제공
- 토론은 과열될 수 있으므로 안전장치가 필요함

타겟 그룹:

- 자신의 시사 지식이 확실한지 검증하고 싶은 사람
- 토론을 통해 자신의 논리력, 정보력, 설득력을 확인하고 싶은 사람
- 토론에서 이겼다는 느낌, 내가 알고 있다는 우월감, 성장감을 느끼고 싶은 20~50대
- 시사, 사회, 경제, 정치, 테크, 문화 이슈에 관심 있는 사용자

MVP 목표:

- 실제 크롤링 완성보다, 서비스 핵심 흐름을 빠르게 보여주는 프로토타입이 우선
- 초기 버전은 mock 데이터를 사용해도 됨
- 단, 나중에 실제 크롤러/API/RSS를 붙일 수 있도록 데이터 구조와 함수 구조를 분리해줘
- 프론트엔드에서 서비스 흐름이 명확하게 보여야 함
- 사용자가 “아 이런 서비스구나”를 바로 이해할 수 있어야 함

기술 조건:

- React 기반으로 만들어줘
- 가능하면 단일 React 컴포넌트 형태로 먼저 만들어줘
- Tailwind CSS를 사용해 깔끔하고 현대적인 UI로 구성해줘
- 백엔드 없이도 동작하는 MVP 프로토타입으로 만들어줘
- 데이터는 우선 mockTrendData 배열로 구성해줘
- 추후 실제 API 연동이 쉽도록 fetchTrends(), startDebate(), generateDebateReply(), judgeDebate() 같은 함수는 분리해줘
- UI 문구는 한국어로 작성해줘

핵심 화면:

1. 메인 대시보드

- 오늘의 실시간 이슈 랭킹
- 카테고리별 이슈 탭
  - 전체
  - 정치/사회
  - 경제
  - 테크
  - 문화/연예
  - 스포츠
  - 커뮤니티
  - 글로벌
- 각 이슈 카드에는 다음 정보 표시
  - 순위
  - 이슈 제목
  - 카테고리
  - 상승률
  - 언급량
  - 감정 분위기: 긍정/부정/중립/논쟁적
  - 관련 키워드
  - 요약 설명
  - 출처 개수
  - “토론 시작” 버튼

2. 이슈 상세 패널

- 사용자가 이슈를 클릭하면 오른쪽 또는 모달로 상세 정보 표시
- 상세 정보에는 다음 항목 포함
  - 이슈 요약
  - 왜 지금 뜨는지
  - 핵심 쟁점
  - 찬성 측 주요 논리
  - 반대 측 주요 논리
  - 관련 키워드
  - 위험도 또는 논쟁성 점수
  - 출처 목록 mock 표시

3. 토론 시작 설정

- 사용자가 이슈를 선택하고 토론 시작 가능
- 토론 시작 전 선택 옵션
  - 내가 찬성 입장
  - 내가 반대 입장
  - AI가 내 의견을 듣고 반대 입장 자동 선택
  - 난이도: 쉬움 / 보통 / 어려움
  - 토론 라운드 수: 3 / 5 / 7
- 시작 버튼을 누르면 토론방으로 이동

4. AI vs 사용자 토론방

- 화면 구성:
  - 상단: 이슈 제목, 현재 라운드, 난이도, 과열도 표시
  - 중앙: 채팅형 토론 메시지
  - 하단: 사용자 입력창
- 토론 AI는 다음 역할을 수행
  - 사용자의 주장에 반박
  - 근거 요구
  - 논리적 허점 지적
  - 다른 관점 제시
  - 너무 공격적이지 않게 토론 유지
- 사용자가 메시지를 입력하면 mock AI 응답이 생성되도록 구현
- 실제 LLM API는 붙이지 말고, 함수 구조만 LLM 연결 가능하게 만들어줘
- 예:
  generateDebateReply(issue, userMessage, userStance, difficulty, roundHistory)

5. 과열 방지 시스템

- 토론 중 사용자의 표현이 공격적이면 과열도 증가
- 과열도는 0~100 점
- 다음 기준으로 동작:
  - 0~39: 정상
  - 40~69: 주의
  - 70 이상: 과열
- 과열 시 UI에서 경고 표시
- 과열도가 높으면 AI가 다음과 같이 반응
  - “표현은 강하지만, 논점을 분리해서 다시 정리해보겠습니다.”
  - “인신공격보다는 근거 중심으로 이어가보겠습니다.”
- 욕설, 혐오, 개인정보 공격, 폭력 조장, 허위사실 단정은 감점 요소로 기록
- 토론을 강제로 막지는 말되, MVP에서는 경고와 피드백으로 처리

6. 토론 종료 및 판단 AI 리포트

- 정해진 라운드가 끝나거나 사용자가 “토론 종료”를 누르면 결과 리포트 표시
- 판단 AI는 중립적으로 분석
- 리포트에는 다음 포함
  - 전체 요약
  - 사용자 핵심 주장
  - AI 핵심 반론
  - 사용자 논리력 점수
  - 근거 활용 점수
  - 반박 대응 점수
  - 감정 조절 점수
  - 설득력 점수
  - 사용자가 잘한 점
  - 보완할 점
  - 논리적 오류 가능성
  - 다음 토론에서 개선할 팁
  - 최종 판정: 사용자 우세 / AI 우세 / 팽팽함
- 판정은 자극적이지 않게 하되, 사용자가 성취감과 재도전 욕구를 느끼도록 작성
- 예:
  “사용자는 핵심 쟁점을 빠르게 잡았지만, 근거 출처 제시가 부족했습니다.”
  “감정적 표현이 일부 있었으나, 주장의 방향성은 명확했습니다.”
  “다음에는 수치, 사례, 반대 측 논리 선제 대응을 추가하면 더 강한 토론이 됩니다.”

7. 기록 화면

- MVP에서는 localStorage를 사용해 토론 결과 저장
- 최근 토론 기록 목록 표시
- 항목:
  - 이슈 제목
  - 날짜
  - 결과
  - 총점
  - 다시 보기 버튼

Mock 데이터:

- 최소 10개 이상의 이슈 데이터를 만들어줘
- 실제 특정 인물 비방이나 민감한 허위사실처럼 보일 수 있는 내용은 피하고, 일반적인 형태의 이슈로 만들어줘
- 예시 이슈:
  - AI 규제 논쟁
  - 전기차 보조금 축소
  - 주 4일제 도입 논쟁
  - 청년 주거 정책
  - 플랫폼 수수료 논란
  - 생성형 AI 저작권 문제
  - 학교 스마트폰 제한
  - 원격근무 축소
  - 물가 상승 체감
  - 스포츠 스타 이적설

디자인 방향:

- 어두운 배경 기반의 프리미엄 느낌
- 실시간 이슈 서비스처럼 빠르고 생동감 있게
- 카드형 UI
- 랭킹 숫자가 잘 보여야 함
- 상승률, 언급량, 논쟁성 점수가 시각적으로 보여야 함
- 토론방은 채팅 UI처럼 구성
- 판단 리포트는 리포트 카드처럼 신뢰감 있게 구성
- 사용자가 “내 논리력이 몇 점인지” 바로 볼 수 있게 점수 UI를 강조
- 너무 장난스럽지 않게, 하지만 승부욕을 자극하는 느낌

MVP에서 반드시 구현할 기능:

- 이슈 랭킹 목록 표시
- 카테고리 필터
- 이슈 상세 보기
- 토론 시작 설정
- AI vs 사용자 토론 채팅
- mock AI 반론 생성
- 과열도 계산 및 경고
- 토론 종료
- 판단 AI 리포트 생성
- localStorage에 토론 기록 저장
- 토론 기록 목록 표시
- 새 토론 시작

중요한 UX:

- 사용자가 처음 들어오면 바로 “지금 뜨는 이슈”가 보여야 함
- 이슈를 클릭하면 상세 내용이 즉시 보여야 함
- “토론 시작” 버튼이 눈에 잘 보여야 함
- 사용자는 복잡한 설정 없이 바로 토론할 수 있어야 함
- 토론이 끝나면 결과 리포트가 자동으로 떠야 함
- 점수와 피드백은 사용자가 공유하고 싶을 정도로 깔끔해야 함

AI 역할 설계:

1. 트렌드 정리 AI

- 여러 출처의 이슈를 짧고 명확하게 요약
- 왜 뜨는지 설명
- 찬반 논점을 정리

2. 토론 AI

- 사용자의 반대편에서 논리적으로 반박
- 사용자의 주장에 근거가 부족하면 근거 요청
- 사용자의 주장이 감정적이면 논점으로 되돌림
- 너무 공격적이지 않게 유지
- 사용자의 지식 검증을 돕는 코치 역할도 일부 수행

3. 판단 AI

- 토론 종료 후 중립 평가
- 사용자의 논리력, 근거 활용, 반박 대응, 감정 조절, 설득력을 점수화
- 사용자가 잘한 점과 개선점을 알려줌
- 최종 판정 제공

안전장치:

- 토론이 과열될 수 있으므로 다음 기능을 포함
  - 과열도 점수
  - 공격적 표현 감지
  - 인신공격 경고
  - 근거 중심 토론 유도
  - 혐오, 폭력, 개인정보 공격, 허위사실 단정 방지
- 정치/사회 이슈는 단정적인 사실 판단을 피하고 “주장”, “논쟁”, “가능성”, “보도에 따르면” 같은 표현을 사용
- 판단 AI는 사용자를 조롱하거나 모욕하지 않음
- 사용자가 이겼다는 느낌은 줄 수 있지만, 허위 우월감보다는 “논리적 성장감” 중심으로 설계

코드 요구사항:

- React 컴포넌트로 작성
- 상태 관리는 useState, useMemo, useEffect 정도로 구현
- 별도 백엔드 없이 동작
- mock 데이터 포함
- localStorage 저장 구현
- 함수 분리:
  - fetchTrends()
  - calculateTrendScore()
  - getSentimentLabel()
  - startDebateSession()
  - generateDebateReply()
  - calculateHeatScore()
  - judgeDebate()
  - saveDebateHistory()
  - loadDebateHistory()
- 나중에 실제 API 연결이 쉽도록 주석으로 API 연결 위치 표시
- 모바일에서도 볼 수 있게 반응형 구성
- 에러 없이 실행 가능한 코드로 작성

출력 형식:

1. 먼저 전체 설계 요약을 짧게 설명
2. 그 다음 바로 코드 제공
3. 코드 아래에 향후 백엔드 연결 시 필요한 API 목록을 정리
4. 마지막으로 다음 단계 개발 로드맵을 제안

중요:

- 단순 설명만 하지 말고 실제 프로토타입 코드를 만들어줘
- UI가 실제 서비스처럼 보여야 함
- MVP 범위를 넘는 과도한 기능은 넣지 말고, 핵심 흐름에 집중해줘
- 하지만 추후 확장 가능한 구조로 작성해줘
