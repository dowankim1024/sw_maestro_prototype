# 이슈캐스트 제품 기획 문서

> 기준 문서: `refactor/01_product_structure.md`, `refactor/02_character_persona_prompts.md`
> 이전 TrendArena/세대·성격 프레임 문서는 최신 제품 방향에 맞춰 이슈캐스트/4렌즈 구조로 정리했다.

---

## 1. 한 줄 정의

**같은 이슈를 4명의 시각으로 30초 안에 짚어보고, 더 궁금하면 그 시각의 캐릭터와 가볍게 대화하는 모바일 이슈 앱.**

---

## 2. 핵심 사용자

| 우선순위 | 사용자 | 제품에서 얻는 가치 |
|---|---|---|
| 1순위 | 정보 깊이 추구자 | 4명의 시각 비교, 캐릭터 대화, "오늘 새로 본 것" 누적으로 이슈를 더 잘 이해한다. |
| 2순위 | Talking Point 추구자 | 30초 카드만 보고도 "오늘 이거 알아" 수준의 대화 소재를 얻는다. |

핵심 wedge는 정보 깊이 추구자다. 단순히 빠르고 트렌디한 피드가 아니라, **가벼운 깊이**와 **다른 정보 레이어**를 제공하는 데 제품 정체성이 있다.

---

## 3. 제품 원칙

1. **비논쟁 노선**: 찬반, 점수, 등급, 자유 댓글, AI 대립 모드는 두지 않는다.
2. **사실 우선**: 핵심 사실과 출처를 먼저 보여준 뒤 캐릭터 시각을 보여준다.
3. **팩트와 의견 분리**: 팩트/보도됨/불확실/캐릭터 의견 라벨을 UI에서 명확히 구분한다.
4. **동적 면책**: 민감 이슈에 더 강한 면책을 표시하고, 모든 이슈에 같은 경고를 남발하지 않는다.
5. **모바일 우선**: 카드와 릴스형 탐색을 중심으로 30초 소비를 지원한다.
6. **짧은 대화**: character.ai식 끝없는 대화가 아니라 3-4턴 후 자연스러운 저장/마무리로 이어진다.

---

## 4. 4가지 렌즈와 캐릭터

기존 저장 키는 데이터 손실을 막기 위해 유지한다. 의미만 최신 캐릭터로 재정의한다.

| 저장 ID | 캐릭터 | 렌즈 | 핵심 질문 |
|---|---|---|---|
| `kkang` | 민철 | 생활자 | 내 일상에 뭐가 닿나? |
| `uncle` | 수진 교수 | 전문가 | 구조적으로 어떻게 작동하나? |
| `prof` | 지오 | 트렌드 | 지금 사람들이 어떻게 반응하나? |
| `pm` | 도윤 | 회의주의자 | 이거 진짜야? 다른 면은 없나? |

이 4명은 서로 논쟁하지 않는다. 자기 영역 밖의 질문은 더 적합한 캐릭터에게 양보한다.

## 5. 주요 화면

### 홈

- 오늘의 이슈 목록
- 카테고리 필터
- 30초 소비 라벨
- 본 이슈 다시 보기 표시
- 릴스 모드 진입

### 이슈 상세

1. 제목과 한 줄 요약
2. 민감 이슈 면책
3. 핵심 사실 3개와 출처 라벨
4. 왜 지금 뜨는지
5. 같은 이슈, 4명의 시각
6. 빠른 반응
7. 출처 목록

### 대화

- 캐릭터별 관점 미리보기
- 추천 질문 칩
- 빠른 프롬프트
- 출처 토글
- "오늘 새로 본 것" 저장

### 컬렉션

- 대화 후 생성된 인사이트 카드 누적
- 캐릭터별 필터
- 카드 다시 보기, 삭제, 전체 비우기

### 릴스 모드

- 세로 스와이프: 다른 이슈
- 가로 스와이프: 표지 → 4명 시각 → 핵심 사실
- 데스크톱 키보드 대응

## 6. 데이터 구조

현재 MVP는 mock 데이터 6건을 사용한다.

- `Issue`: 제목, 요약, 왜 지금, 카테고리, 키워드, 안전도, 커버
- `IssueFact`: 핵심 사실, 출처 ID, 신뢰 라벨
- `Source`: 출처 제목, 발행처, URL, 발행/조회 시각
- `CharacterAngle`: 캐릭터별 one-liner, viewpoint, 의견 면책, 연결된 사실 ID
- `ChatTurn`: 사용자/캐릭터/시스템 대화 내역
- `InsightCard`: 대화 종료 후 저장되는 "오늘 새로 본 것"

모든 캐릭터 관점은 `referencedFactIds`를 통해 실제 사실 카드에 연결된다.

---

## 7. LLM 설계

### 캐릭터 답변

- 서버 Route Handler(`/api/chat/reply`)에서 Gemini를 호출한다.
- 시스템 프롬프트는 공통 가드레일 + 캐릭터별 페르소나로 구성한다.
- 사용자 메시지에는 이슈 정보, 핵심 사실, 출처 라벨, 대화 히스토리를 포함한다.
- API 키가 없거나 실패하면 캐릭터 톤의 로컬 폴백으로 응답한다.

### 인사이트 카드

- 서버 Route Handler(`/api/chat/insight`)에서 별도 JSON 생성 프롬프트를 사용한다.
- 캐릭터 발언을 그대로 복사하지 않고 사용자가 새로 본 시각으로 재진술한다.
- 대화와 핵심 사실에 없는 숫자, 인물, 기관, 출처는 만들지 않는다.

---

## 8. 저장 정책

MVP는 서버 계정 없이 `localStorage`를 사용한다.

- `issuecast.cards.v1`: 인사이트 카드
- `issuecast.reactions.v1`: 빠른 반응
- `issuecast.seen.v1`: 본 이슈 ID

리팩토링 후에도 기존 키를 유지해 저장 데이터가 유실되지 않게 한다.

---

## 9. KPI

카드 조회수보다 깊이 추구자의 행동을 우선한다.

- 대화 진입율
- "오늘 새로 본 것" 저장 수
- 같은 이슈의 다른 캐릭터 재진입
- 컬렉션 재방문 빈도

---

## 10. 의도적으로 제외하는 것

| 제외 항목 | 이유 |
|---|---|
| 찬반 대립 | 비논쟁 노선과 충돌 |
| 점수·레벨·랭킹 | 지식 경쟁으로 흐를 위험 |
| 자유 댓글 | 사람 간 논쟁 유발 가능성 |
| 거시 렌즈 | 트렌드성 이슈에서 어색해 회의주의자 렌즈로 대체 |
| 출처 없는 AI 사실 생성 | 환각 리스크 |

---

## 11. 운영 메모

- 신규 이슈 추가 시 4명 모두 자연스럽게 작동하는지 확인한다.
- 모든 이슈는 최소 2개 이상의 출처를 가져야 한다.
- 민감 이슈는 `safetyLevel`을 `sensitive` 이상으로 지정한다.
- `prompt/` 폴더의 과거 프롬프트는 레거시 추적용이며, 현재 런타임은 `src/lib/prompts.ts`를 사용한다.
